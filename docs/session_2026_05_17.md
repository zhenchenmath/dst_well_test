## What was discussed and decided

### 1. Starting point
Read CLAUDE.md (original plan: three Julia scripts for three DST physics cases).
User had modifications before any code was written.

### 2. Language split
Original plan was all-Julia. User changed this: Python for everything except simulation.
Julia is only used because JutulDarcy has no Python equivalent.
Interface: file-based I/O (JSON config + HDF5 fields in, HDF5 results out).
juliacall/PyJulia considered and rejected (too complex for a sandbox).

### 3. Python wrapper design
User wanted a `SimulationModel` class with a `.simulate()` method — like a neural network
model you call `.predict()` on. Analogy to PhysicsNeMo (NVIDIA) for ML experiments.
Config in YAML, results accessible in Python/Jupyter.

### 4. Config system
YAML + pydantic v2. Modeled after ML experiment configs (PhysicsNeMo/Hydra style).
One config = one run. No sweep/grid-search feature.
Cross-examination (overlaying multiple runs) done in notebook via `load_experiment()`.

### 5. Experiment/run hierarchy
Two-level: `experiment/run`. Flat structure rejected as too massive after many runs.
Experiment = the physics question. Run = base case + variations.
Outputs: `outputs/{experiment}/{run}/`. Configs mirror: `configs/{experiment}/{run}.yaml`.

### 6. Field generation design
User wanted perm/poro field generation separate from simulation — treated as a subproject.
Future plan: interactive 2D drawing tool to paint perm fields and well locations.
Decision: always require explicit `.h5` file in config (no inline scalar shorthand).

Three templates requested:
- homogeneous (baseline)
- fourier (spectral synthesis, spatially correlated log-normal)
- gmm (facies-based, Truncated Gaussian Simulation)

Scaling discussion: log-normal for perm (standard in RE), perm_std_log parameter for
heterogeneity level (0.5=mild, 1.0=moderate, 2.0=strong). Porosity clipped [0.03, 0.45].

### 7. Well location
Decided: boolean mask in field `.h5` (not coordinates in YAML).
Reason: aligns with future interactive drawing tool; natural output of a paint operation.

### 8. PTA analysis
Bourdet derivative: implement from scratch in NumPy (pedagogically valuable, ~30 lines).
welltestpy: install as comparison library. User only needs Bourdet, not full PTA suite.

### 9. Visualization
Defer detailed visualization discussion to later session.
Outputs: files in `outputs/`, notebook or HTML for interactive cross-examination.
